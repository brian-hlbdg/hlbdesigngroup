The Problem

Transportation operations span multiple time zones, with drivers, dispatchers, and customers all operating in different geographic locations. The original system displayed all times in a single time zone, leading to confusion, missed pickups, and scheduling errors.

Key challenges included:

  • Drivers seeing appointment times in the wrong time zone
  • Dispatchers manually calculating time differences
  • Customers receiving confusing delivery estimates
  • Daylight saving time transitions causing system errors
  • No real-time updates when schedules changed

Design Solution

I designed a contextual time component that automatically detects and displays relevant time zones based on the user's role and the specific operation. The component shows multiple time zones simultaneously when needed, with clear labeling and visual hierarchy.

User Testing & Iterations

I conducted testing sessions with drivers, dispatchers, and customer service representatives to validate the design approach. Key insights from testing:

  • Users needed to see "their" time zone prominently displayed
  • Color coding helped distinguish between different time zones
  • Relative time indicators ("in 2 hours") were more useful than absolute times
  • Mobile users needed larger touch targets and simplified layouts

Technical Implementation

The component is built as a Phoenix LiveView module that handles real-time time zone calculations server-side, ensuring accuracy and consistency across all clients.

lib/tms_web/live/components/time_component.ex 
defmodule TMSWeb.Components.TimeComponent do
                    use TMSWeb, :live_component

                    def render(assigns) do
                      ~H"""
                      <div class="time-zone-display">
                        <%= for {zone, label} <- @time_zones do %>
                          <div class={["time-zone-card", @active_zone == zone && "active"]}>
                            <div class="time-zone-label"><%= label %></div>
                            <div class="time-value">
                              <%= format_time(@current_time, zone) %>
                            </div>
                            <div class="date-value">
                              <%= format_date(@current_time, zone) %>
                            </div>
                          </div>
                        <% end %>
                      </div>
                      """
                    end

                    def mount(socket) do
                      # Update time every second for real-time display
                      if connected?(socket) do
                        :timer.send_interval(1000, self(), :tick)
                      end

                      socket = assign(socket, current_time: DateTime.utc_now())
                      
                      {:ok, socket}
                    end

                    def handle_info(:tick, socket) do
                      socket = assign(socket, current_time: DateTime.utc_now())
                      {:noreply, socket}
                    end

                    defp format_time(datetime, timezone) do
                      datetime
                      |> DateTime.shift_zone!(timezone)
                      |> Calendar.strftime("%I:%M %p")
                    end

                    defp format_date(datetime, timezone) do
                      datetime
                      |> DateTime.shift_zone!(timezone)
                      |> Calendar.strftime("%A, %B %d")
                    end
                  end

Integration Example

Here's how the time component is integrated into the main TMS interface:

Integration in TMS LiveView 
# In the main TMS dispatch view
                    def render(assigns) do
                      ~H"""
                      <div class="dispatch-dashboard">
                        <.live_component 
                          module={TMSWeb.Components.TimeComponent}
                          id="route-times"
                          time_zones={@route_time_zones}
                          active_zone={@user_timezone}
                          current_time={@current_time} />
                          
                        <div class="route-details">
                          <!-- Route information displayed here -->
                        </div>
                      </div>
                      """
                    end

                    # Automatically determine relevant time zones
                    defp get_route_time_zones(route) do
                      [
                        {route.pickup_timezone, "Pickup Location"},
                        {"America/Chicago", "Dispatch"},
                        {route.delivery_timezone, "Delivery Location"}
                      ]
                      |> Enum.uniq_by(fn {tz, _label} -> tz end)
                    end

Lessons Learned

This component taught me the importance of understanding real-world user contexts. Time zones seem simple until you're dealing with drivers crossing multiple zones, daylight saving time transitions, and the cognitive load of constant mental calculations.

The Phoenix LiveView approach proved ideal for this use case—the server-side time calculations ensure accuracy, while the real-time updates keep everyone synchronized without complex client-side JavaScript.

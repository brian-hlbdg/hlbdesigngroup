Project Overview

The Challenge

NFI Industries' legacy Transportation Management System (TMS) was struggling with usability issues that were impacting productivity and increasing training time for new users. The system's complexity was leading to:

  • High onboarding times of weeks for new employees
  • Frequent user errors leading to operational inefficiencies
  • Anitquated work flows which lead to slow interactions
  • Limited visibility into critical logistics data

Project Goals

I was tasked with creating a comprehensive UX design of a TMS interface with the following goals:

  • Reduce onboarding time by at least 40%
  • Decrease user errors by 30% or more
  • Create a consistent UI pattern library for the entire system
  • Improve overall user efficiency and satisfaction

Research & Discovery

Understanding User Needs

To fully understand the challenges faced by transportation professionals, I conducted extensive user research that included:

User Interviews

I interviewed 12 dispatchers, 5 logistics managers, and 3 IT support staff to understand their workflows, pain points, and feature priorities.

"I spend too much time searching for information across different screens. If I could see all the relevant shipment data in one place, it would save me at least an hour each day." — Senior Dispatcher

Contextual Inquiry

I observed dispatchers in their natural work environment for 35+ hours, noting specific pain points, workarounds, and inefficiencies in their daily workflows.

Observation finding: Users were creating manual cross-references between systems because of poor data visibility in the TMS.

Usability Audit

I conducted a comprehensive audit of the existing system, identifying key usability issues:

Navigation Issues
  • Inconsistent menu structures
  • Excessive click depths (5+ clicks)
  • Poor information hierarchy
UI & Data Presentation
  • Cluttered table displays with excess data
  • Poor contrast and readability
  • No visual hierarchy for critical info
Workflow Issues
  • Disjointed multi-step processes

  • Inefficient data entry requirements

Design Process

Based on our research findings, I developed a systematic approach to redesigning the TMS interface. The process was collaborative and iterative, with regular feedback from end users and stakeholders.

1. Information Architecture Redesign

I conducted a complete restructuring of the system's information architecture, focusing on logical grouping and intuitive navigation patterns. This involved:

  • Card sorting exercises with dispatchers
  • Task flow analysis and optimization
  • Navigation menu restructuring

Key Outcome

Reduced the navigation depth from 5+ clicks to a maximum of 3 clicks for any common task, and created logical groupings that matched dispatchers' mental models.

2. Wireframing & Interface Design

I created detailed wireframes for key screens, focusing on clarity, visual hierarchy, and efficient workflows. The design emphasized:

  • Clear visual hierarchy for critical information
  • Contextual tools and actions
  • Progressive disclosure for complex functions
  • Streamlined data entry with smart defaults

The Solution

The redesigned TMS interface featured several key improvements that directly addressed the pain points identified in our research.

1. Unified Dashboard

The new dashboard consolidated critical information into a single view, featuring dynamic status indicators, priority-based sorting, and customizable views. This gave dispatchers immediate visibility into their most important shipments.

2. Streamlined Shipment Management

The shipment detail view was completely redesigned to show all relevant information in a single screen, with contextual actions and smart filters. This reduced the need for multiple screens and dramatically improved efficiency.

3. Intelligent Exception Handling

A new exception handling system automatically detected and categorized issues, providing guided resolution workflows. This reduced the time and cognitive load required to handle shipment problems.

4. Interactive Planning Tools

New drag-and-drop planning tools made route and schedule adjustments intuitive. Real-time validation prevented errors, and visual indicators highlighted optimization opportunities.

Technical Implementation

As the lead frontend developer on the project, I worked closely with the backend team to implement the new interface using:

Frontend Technologies

  • Phoenix LiveView for reactive UI
  • Tailwind CSS for styling
  • LiveView Components for reusability
  • Alpine.js for enhanced interactions

Development Approach

  • Component-first methodology
  • Storybook for component testing
  • Continuous integration pipeline
  • Phased deployment approach

Performance Optimizations

  • Lazy loading for large datasets
  • Optimized database queries
  • Client-side caching for frequently accessed data
  • Progressive enhancement for critical features

Accessibility Considerations

  • WCAG 2.1 AA compliance throughout the interface
  • Keyboard navigation for all interactive elements
  • High contrast mode for visually impaired users
  • Screen reader optimization with ARIA attributes

While our primary user base didn't have significant accessibility needs, we prioritized accessibility to ensure the system was usable by all employees, regardless of ability.

This forward-thinking approach also ensured compliance with accessibility regulations and prepared the platform for future expansion to external users.

Reflections & Lessons Learned

This project reinforced several key principles and provided valuable lessons that I've carried forward to subsequent work.

What Worked Well

  • Extensive User Research

    Investing time upfront to deeply understand user needs and workflows paid off in a solution that truly addressed pain points.

  • Component-Based Approach

    Building a comprehensive design system accelerated development and ensured consistency across the application.

  • Iterative Testing

    Regular usability testing throughout the process caught issues early and validated our design decisions with real users.

Challenges & Learnings

  • Balancing Complexity vs. Simplicity

    Finding the right balance between powerful functionality and intuitive simplicity was challenging. I learned to use progressive disclosure more effectively.

  • Data Visualization Scale

    We initially underestimated the complexity of representing large datasets. In future projects, I'd allocate more time to this aspect of the design.

  • Change Management

    While the new interface was objectively better, we faced initial resistance from long-time users. Earlier involvement of power users in the design process would have helped.

Key Takeaways

This project highlighted several principles that I now apply to all of my UX work:

Context is Everything

Understanding the full context of use—including environment, constraints, and user goals—is essential for creating truly effective interfaces. There's no substitute for observing users in their actual working environment.

Systems, Not Screens

Approaching design as a complete system rather than a collection of individual screens leads to more coherent, usable products. The component-based approach we used was key to the project's success.

Measure What Matters

Clear metrics tied to business goals help drive design decisions and demonstrate value. By establishing metrics early and tracking them throughout, we were able to show the concrete impact of our work.
